Introduction to Wireshark

Wireshark is the world's foremost and widely-used network protocol analyzer. It lets you see what's happening on your network at a microscopic level and is the de facto (and often de jure) standard across many commercial and non-profit enterprises, government agencies, and educational institutions.

Why Learn Wireshark?

Network troubleshooting, security analysis, software and protocol development, and education are just a few of the many uses for Wireshark. In the cybersecurity field, it's an essential tool for understanding network communications and identifying potential security threats.

Key Features:

  • Live capture and offline analysis: Capture packets in real-time or analyze previously captured data
  • Deep inspection: Examine hundreds of protocols with more being added constantly
  • Rich VoIP analysis: Comprehensive Voice over IP analysis capabilities
  • Cross-platform: Runs on Windows, Linux, macOS, and other platforms
  • Open source: Free to use with active community development

Installation Guide

System Requirements:

  • Windows 10/11, macOS 10.14+, or Linux distributions
  • Minimum 2GB RAM (4GB recommended)
  • 500MB free disk space
  • Network interface card (NIC)

Installation Steps:

Windows Installation

1. Visit wireshark.org/download.html

2. Download the Windows installer

3. Run as administrator

4. Follow the installation wizard

5. Install Npcap when prompted

Linux Installation

Ubuntu/Debian:

sudo apt update
sudo apt install wireshark

CentOS/RHEL:

sudo yum install wireshark

macOS Installation

1. Download the macOS disk image

2. Open the .dmg file

3. Drag Wireshark to Applications

4. Install ChmodBPF for packet capture

Important Note

For packet capturing on Windows, you'll need to install Npcap (recommended) or WinPcap. Npcap is the modern packet capture library and is included with recent Wireshark installations.

User Interface Overview

Understanding Wireshark's interface is crucial for effective packet analysis. The interface is divided into several key areas:

Menu Bar & Toolbar

Contains all the main functions including File operations, Edit functions, View settings, and analysis tools. The toolbar provides quick access to frequently used features.

Packet List Pane

Displays captured packets in chronological order. Shows time, source, destination, protocol, length, and basic information for each packet.

Packet Details Pane

Shows detailed protocol information for the selected packet. You can expand different protocol layers to see specific fields and values.

Packet Bytes Pane

Displays the raw data of the selected packet in both hexadecimal and ASCII format. Useful for low-level analysis.

Key Interface Elements:

  • Filter Bar: Apply display filters to show specific packets
  • Status Bar: Shows capture statistics and filter information
  • Column Headers: Click to sort packets by different criteria
  • Protocol Tree: Navigate through protocol layers in the details pane

Capturing Packets

Starting a Capture:

  • Launch Wireshark with administrator/root privileges
  • Select the network interface you want to monitor
  • Click the start button (shark fin icon) or press Ctrl+E
  • Packets will begin appearing in real-time
# Command line capture example sudo wireshark -i eth0 -k # -i: specify interface # -k: start capturing immediately

Capture Options:

Interface Selection

Choose the correct network interface:

  • Ethernet interfaces (eth0, ens33)
  • WiFi interfaces (wlan0, wlp2s0)
  • Loopback interface (lo)
  • USB interfaces

Capture Filters

Set filters before capturing to limit data:

  • host 192.168.1.1
  • port 80 or port 443
  • tcp and not ssh

Capture Duration

Control capture length:

  • Set time limits
  • Set packet count limits
  • Set file size limits
  • Ring buffer options

Ethical Considerations

Always ensure you have proper authorization before capturing network traffic. Only capture traffic on networks you own or have explicit permission to monitor. Unauthorized packet capture may violate laws and regulations.

Applying Filters

Filters are essential for managing large amounts of captured data. Wireshark offers two types of filters:

Display Filters:

Applied after capture to show/hide specific packets in the packet list.

Protocol Filters

http
tcp
dns
ssl
icmp

IP Address Filters

ip.addr == 192.168.1.1
ip.src == 10.0.0.1
ip.dst == 8.8.8.8
net 192.168.0.0/24

Port Filters

tcp.port == 80
udp.port == 53
tcp.srcport == 443
tcp.dstport == 22

Logical Operators

http and ip.addr == 192.168.1.1
tcp.port == 80 or tcp.port == 443
not ssh
(http or https) and ip.src == 10.0.0.1

Capture Filters:

Applied before capturing to limit what packets are captured (uses Berkeley Packet Filter syntax).

# Capture only HTTP traffic port 80 # Capture traffic to/from specific host host 192.168.1.100 # Capture only TCP traffic tcp # Exclude SSH traffic not port 22

Filter Tips

Use the filter expression builder (Expression button) for complex filters. Wireshark also provides autocomplete suggestions as you type. Save frequently used filters for quick access.

Protocol Analysis

Analyzing Different Protocols:

HTTP Analysis

Examine web traffic:

  • HTTP requests and responses
  • Headers and cookies
  • Status codes
  • Content types
http.request.method == "GET"

HTTPS/TLS Analysis

Secure communications:

  • TLS handshake process
  • Certificate information
  • Cipher suites
  • Encrypted data streams
tls.handshake.type == 1

DNS Analysis

Domain name resolution:

  • DNS queries and responses
  • Record types (A, AAAA, MX, etc.)
  • Response codes
  • TTL values
dns.qry.name contains "example.com"

TCP Analysis

Connection-oriented protocol:

  • Three-way handshake
  • Sequence numbers
  • Window sizes
  • Connection termination
tcp.flags.syn == 1

Analysis Features:

  • Follow TCP Stream: View complete conversations between hosts
  • Protocol Statistics: Get overview of protocol distribution
  • IO Graphs: Visualize traffic patterns over time
  • Expert Information: Automatic detection of network issues

Following Streams

Right-click on a packet and select "Follow" → "TCP Stream" to see the entire conversation. This is particularly useful for analyzing application-layer protocols like HTTP, FTP, or Telnet.

Exporting Data

Export Options:

Packet Data

Export captured packets:

  • PCAP/PCAPNG format
  • Selected packets only
  • Filtered packets
  • Specific packet ranges

CSV Format

Export to spreadsheet:

  • Packet summary information
  • Custom field selection
  • Statistical data
  • Time series data

Objects & Files

Extract embedded content:

  • HTTP objects
  • Images and media files
  • Documents
  • Executables

Export Steps:

  • Go to File → Export Specified Packets
  • Choose the file format and location
  • Select packet range (all, selected, or filtered)
  • Configure export options
  • Click Save to export
# Command line export examples tshark -r input.pcap -w output.pcap "http" tshark -r input.pcap -T csv > output.csv tshark -r input.pcap -T json > output.json

Best Practices

Legal & Ethical

  • Only capture on authorized networks
  • Respect privacy and confidentiality
  • Follow company policies
  • Comply with local laws
  • Document authorization

Technical Tips

  • Use capture filters for large networks
  • Save captures regularly
  • Use ring buffers for long captures
  • Monitor system resources
  • Validate findings with multiple sources

Data Management

  • Organize capture files systematically
  • Use descriptive file names
  • Include timestamps and locations
  • Backup important captures
  • Delete sensitive data when appropriate

Analysis Workflow

  • Start with broad overview
  • Use filters to narrow focus
  • Follow protocols layer by layer
  • Document findings
  • Cross-reference with other tools

Continuous Learning

Wireshark is a powerful tool with many advanced features. Regular practice, staying updated with new versions, and participating in the cybersecurity community will help you master network analysis skills.

Practical Assignments

Complete these hands-on assignments to reinforce your Wireshark learning. Each task is designed to build upon previous knowledge and provide practical experience with real-world scenarios.

1
Basic Packet Capture and Interface Familiarization
Beginner
Objective: Learn to capture packets and navigate the Wireshark interface.
Steps to Complete:
  1. Install Wireshark on your system following the installation guide
  2. Launch Wireshark with administrator privileges
  3. Identify and select your active network interface
  4. Start a packet capture for 2-3 minutes
  5. Browse different websites (google.com, youtube.com, etc.)
  6. Stop the capture and explore the three main panes
  7. Click on different packets to see how information changes
  8. Take screenshots of your capture showing different protocol types
Expected Results:
  • Successfully captured network traffic
  • Screenshots showing HTTP, TCP, DNS, and other protocols
  • Understanding of packet list, details, and bytes panes
  • Ability to identify source/destination IPs and ports
Tips:
  • Use a wired connection if possible for cleaner captures
  • Close unnecessary applications to reduce network noise
  • Try right-clicking on packets to explore context menu options
2
HTTP Traffic Analysis
Beginner
Objective: Analyze HTTP traffic and understand web communication protocols.
Steps to Complete:
  1. Start a new packet capture
  2. Visit a non-HTTPS website (like neverssl.com or example.com)
  3. Apply the display filter: http
  4. Find HTTP GET and POST requests
  5. Right-click on an HTTP packet and select "Follow TCP Stream"
  6. Analyze the HTTP headers, user-agent, and response codes
  7. Export the HTTP objects from File → Export Objects → HTTP
  8. Document your findings about the communication
Expected Results:
  • Identification of HTTP request methods (GET, POST)
  • Understanding of HTTP status codes (200, 404, etc.)
  • Analysis of HTTP headers (Host, User-Agent, Content-Type)
  • Successfully exported web objects
Tips:
  • Many modern sites use HTTPS, so find HTTP-only sites for this exercise
  • Look for images, CSS, and JavaScript files in the HTTP objects
  • Pay attention to cookies and session information
3
DNS Query Investigation
Beginner
Objective: Understand DNS resolution process and analyze DNS traffic.
Steps to Complete:
  1. Clear your DNS cache: ipconfig /flushdns (Windows) or sudo dscacheutil -flushcache (macOS)
  2. Start packet capture
  3. Apply display filter: dns
  4. Visit several different websites
  5. Observe DNS queries and responses
  6. Identify A, AAAA, and CNAME record types
  7. Note the DNS servers being used
  8. Find the response times for DNS queries
Expected Results:
  • Captured DNS queries and responses
  • Identification of different DNS record types
  • Understanding of DNS server addresses
  • Analysis of DNS response codes and timing
Tips:
  • Look for both IPv4 (A) and IPv6 (AAAA) records
  • Check if your system uses your router or public DNS servers
  • Try using nslookup command while capturing to see the difference
4
TCP Three-Way Handshake Analysis
Intermediate
Objective: Analyze TCP connection establishment and termination processes.
Steps to Complete:
  1. Start a fresh packet capture
  2. Apply filter: tcp.flags.syn == 1
  3. Open a web browser and visit a website
  4. Find a TCP SYN packet and note its sequence number
  5. Remove the filter and find the corresponding SYN-ACK and ACK packets
  6. Analyze the sequence and acknowledgment numbers
  7. Create a filter to show connection termination: tcp.flags.fin == 1
  8. Document the complete TCP connection lifecycle
Expected Results:
  • Identification of SYN, SYN-ACK, ACK sequence
  • Understanding of sequence and acknowledgment numbers
  • Analysis of TCP window sizes
  • Documentation of connection establishment and termination
Tips:
  • Use the "Follow TCP Stream" feature to see the complete conversation
  • Look for RST packets which indicate connection resets
  • Compare different connections to the same server
5
Advanced Filtering and Statistics
Intermediate
Objective: Master advanced filtering techniques and generate network statistics.
Steps to Complete:
  1. Capture network traffic for 10 minutes during normal internet usage
  2. Create and apply these complex filters:
    • http.request.method == "GET" and http.host contains "google"
    • tcp.port == 80 or tcp.port == 443
    • ip.addr == [your gateway IP] and not arp
  3. Go to Statistics → Protocol Hierarchy
  4. Generate an I/O Graph (Statistics → I/O Graphs)
  5. Check Statistics → Conversations to see traffic patterns
  6. Export filtered data to CSV format
  7. Create a summary report of your findings
Expected Results:
  • Successful application of complex filters
  • Protocol hierarchy analysis showing traffic distribution
  • I/O graphs showing traffic patterns over time
  • Exported data in CSV format
  • Comprehensive analysis report
Tips:
  • Save useful filters for future use
  • Use the Expression builder for complex filters
  • Experiment with different graph types in I/O Graphs
  • Compare statistics before and after applying filters
6
Network Troubleshooting Scenario
Intermediate
Objective: Use Wireshark to troubleshoot a simulated network issue.
Steps to Complete:
  1. Simulate a network issue by:
    • Trying to access a non-existent website
    • Accessing a website with firewall/proxy issues
    • Connecting to a server on a wrong port
  2. Capture the traffic during these attempts
  3. Use Expert Information (Analyze → Expert Information)
  4. Look for error codes, timeouts, and retransmissions
  5. Apply filters to isolate problem packets:
    • tcp.analysis.flags
    • icmp.type == 3 (Destination Unreachable)
  6. Document the root cause of each issue
  7. Propose solutions for the identified problems
Expected Results:
  • Identification of network issues using packet analysis
  • Understanding of TCP retransmissions and timeouts
  • Analysis of ICMP error messages
  • Troubleshooting report with root causes and solutions
Tips:
  • Pay attention to the Expert Info panel for automatic issue detection
  • Look for patterns in timing and sequence numbers
  • Use the "Time" column to identify delays
  • Check for duplicate ACKs and fast retransmissions
